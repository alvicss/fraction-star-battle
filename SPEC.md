# 分數星際大作戰 — 功能規格文件

**版本：** v1.0
**最後更新：** 2026-03-27
**目標對象：** 國小四年級學生
**技術形式：** 單一 HTML 檔案（純前端，無需伺服器）

---

## 一、故事背景

| 角色 | 符號 | 說明 |
|---|---|---|
| 小勇士（玩家） | 👨‍🚀 | 來自地球的數學勇士 |
| 亂亂怪 | 👾 | 入侵數學星球的宇宙怪獸，愛嗆聲但其實怕輸 |
| FRAKIO | 🤖 | 守護機器人，擔任導師，失敗時給予鼓勵 |

**故事：** 亂亂怪入侵數學星球，把所有分數石碑都打亂。玩家與 FRAKIO 用「分數卡牌決鬥」對抗亂亂怪，依序挑戰四個星球，最終收復數學星球。

---

## 二、整體遊戲流程

```
開場畫面 → 關卡選擇 → 對戰畫面（10 局制） → 結算畫面
```

- 4 個關卡全部開放（無需解鎖），建議依 1→4 順序挑戰
- 每場固定 **10 局**
- 血量歸零時立即結算（不等到第 10 局結束）

## 四、卡牌系統

### 4.1 卡牌類型

| 類型 | 標籤顏色 | 說明 | 範例 |
|---|---|---|---|
| 真分數 | 青綠 | 分子 < 分母（最簡） | 3/5 |
| 假分數 | 紫色 | 分子 > 分母（最簡） | 7/4 |
| 帶分數 | 橘色 | 整數部分 + 真分數 | 2又1/3 |
| 整數牌 | 綠色 | 純整數（關卡3/4B5用） | 4 |

### 4.2 數值限制

| 限制項目 | 規則 |
|---|---|
| 分母範圍 | 2～9（每局隨機，雙方相同） |
| 分子上限 | ≤ 20 |
| 帶分數整數部分 | ≤ 15（且不超過 `floor(19/d)`）|
| 整數牌範圍 | 2～9 |
| 同組 4 張牌 | 禁止數值重複 |

### 4.3 每局發牌

- 每局開始隨機選定一個分母 `d`（2～9）
- **雙方分母相同**（本遊戲不出現不同分母的計算）
- 每局結束後重新發牌（分母可能更換）

## 五、關卡詳細規格

### 關卡 1：加法星球 🪐

**學習目標：** 帶分數、假分數、真分數的同分母加法

**流程：**
1. 怪獸從 4 張牌中自動出 1 張
2. 玩家從 4 張牌中選 1 張
3. 顯示題目：`[怪獸牌] ＋ [玩家牌] ＝ ？`
4. 玩家輸入帶分數答案（整數部分 + 分子 + 分母）
5. 判定勝負：答對 → 怪獸 -1 血；答錯 → 玩家 -1 血

---

### 關卡 2：減法彗星 ☄️

**學習目標：** 帶分數、假分數、真分數的同分母減法

**流程：**
同關卡 1，但題目為 `[較大牌] － [較小牌] ＝ ？`
（系統自動比較大小，避免出現負數結果）

---

### 關卡 3：乘法銀河 🌌

**學習目標：** 分數（真/假/帶分數）× 整數

**特殊規則：**
- 玩家手牌固定包含 **2 張整數牌 + 2 張分數牌**
- 玩家需先選 1 張分數牌，再選 1 張整數牌
- 題目顯示：`[分數牌] × [整數] ＝ ？`
- **答錯不扣玩家血**（鼓勵學習模式）；答對怪獸 -1 血

---

### 關卡 4：比大小黑洞 🕳️

每局隨機選擇「玩法 A」或「玩法 B」（各 50% 機率）

#### 玩法 A：判斷符號

1. 怪獸自動出 1 張牌
2. 玩家選 1 張牌
3. 玩家從 `＞` / `＝` / `＜` 三個按鈕中選擇正確符號
4. 答對 → 怪獸 -1 血；答錯 → 玩家 -1 血

#### 玩法 B：拖曳填框（5 種子類型）

| 子類型 | 格式 | 槽數 | 說明 |
|---|---|---|---|
| B1 | `[玩家框] ＞ [怪獸牌]` | 1 | 拖入比怪獸牌大的牌 |
| B2 | `[玩家框] ＜ [怪獸牌]` | 1 | 拖入比怪獸牌小的牌 |
| B3 | `[框] ＋ [框] ＞ [怪獸牌]` | 2 | 兩張牌之和大於怪獸牌 |
| B4 | `[框] － [框] ＞ [怪獸牌]` | 2 | 兩張牌之差大於怪獸牌 |
| B5 | `[框] × [框] ＞ [怪獸牌]` | 2 | 一張分數 × 一張整數，積大於怪獸牌 |

**B5 特殊規則：**
- 玩家手牌包含 ≥ 2 張整數牌
- 系統確保排列組合中**有贏也有輸**，讓孩子需要思考選擇
- 兩格必須是「一張整數 + 一張分數」，否則提示錯誤

**操作方式：**
桌機：拖曳卡牌至框槽 / 手機或觸控：點擊卡牌依序填槽（再次點擊已放入的牌可取回）

## 六、答案驗證規則

### 計算類（關卡 1 / 2 / 3）

```
輸入：整數部分 w、分子 n、分母 d
正確答案：whole_ans、n_ans、d_ans

步驟 1 — 數值相等（允許約分）
  user_total = w × d + n
  ans_total  = whole_ans × d_ans + n_ans
  同值 ⟺ user_total × d_ans = ans_total × d（交叉相乘，避免浮點誤差）

步驟 2 — 分數部分必須是真分數
  合格 ⟺ n = 0 或 n < d

判定：
  同值 且 合格 → ✅ 答對
  同值 但 n ≥ d → ❌ 提示「分數部分是假分數！」
  不同值 → ❌ 提示一般錯誤
```

**關於約分：**
因為尚未教到約分，系統**不要求**學生約分，但學生若自行約分（例如正確答案 4/6，學生填 2/3）也算答對。

### 比大小類（關卡 4）

- 玩法 A：直接比對數值與所選符號
- 玩法 B：計算框槽內卡牌的運算結果，判斷是否符合不等式

## 七、勝負系統

### 每局

| 情況 | 血量變化 |
|---|---|
| 關卡 1/2/4 答對 | 怪獸 -1 血 |
| 關卡 1/2/4 答錯 | 玩家 -1 血 |
| 關卡 3 答對 | 怪獸 -1 血 |
| 關卡 3 答錯 | **無血量變化**（鼓勵模式）|

答錯後可重試（輸入框保持開放），不額外扣分。

### 整場結算（星星評分）

| 結果 | 星星 | 條件 |
|---|---|---|
| ⭐⭐⭐ | 大勝 | 怪獸血量歸 0 |
| ⭐⭐ | 勝利 | 玩家剩血 > 怪獸剩血 |
| ⭐ | 平局 | 剩血差距 ≤ 1 |
| 無星 | 落敗 | 玩家血量歸 0 |

## 十一、驗收清單

| # | 驗收項目 | 驗收方式 |
|---|---|---|
| 1 | 分數顯示正確（帶分數、假分數、真分數） | 目視各關卡卡牌 |
| 2 | 答案輸入假分數時顯示針對性提示 | 關卡1/2/3 輸入分子 ≥ 分母 |
| 3 | 約分答案被接受（如 4/6 → 輸入 2/3 答對）| 手動計算後輸入約分版本 |
| 4 | 關卡3 答錯不扣玩家血 | 連續答錯觀察血量條 |
| 5 | 關卡4B 拖曳填槽（桌機）| 用滑鼠拖曳卡牌至框框 |
| 6 | 關卡4B 點擊填槽（手機/平板）| 點擊卡牌依序填入框框 |
| 7 | B5 乘法驗證：兩框非一整一分時提示錯誤 | 放兩張分數牌 |
