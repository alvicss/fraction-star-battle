## 一、產品定位

《分數星際大作戰》是一款以宇宙冒險包裝的分數練習遊戲。玩家透過選牌、計算與比較大小，和怪獸「亂亂怪」進行 10 局制對戰，在 5 個主題關卡中練習：

- 同分母分數加法
- 同分母分數減法
- 分數乘整數
- 分數大小比較
- 多步驟選牌與策略思考

遊戲強調：

- 低門檻上手
- 單機即可遊玩，無需後端或安裝
- 每局重新發牌，提升變化性
- 以卡牌視覺輔助分數理解
- 用鼓勵式回饋降低答錯挫折感

## 三、整體遊戲流程

```text
開場畫面
→ 腳色選擇（劍士 / 魔法師 / 弓箭手）
→ 關卡選擇（5 關）
→ 對戰畫面（最多 10 局）
→ 結算畫面
→ 再挑戰 / 換關卡
```

### 3.1 全域流程說明

1. 玩家從開場進入**腳色選擇**，挑選代表腳色。
2. 選擇後進入**關卡選擇**，5 個關卡全部直接開放。
3. 每場對戰初始為玩家 5 血、怪獸 5 血，最多 10 局。
4. 每次 `nextRound()` 開始前檢查是否達成結算條件：
   - 已進行 10 局
   - 玩家血量 `<= 0`
   - 怪獸血量 `<= 0`
5. 結算後依剩餘血量差給予 0 到 3 顆星。
6. 關卡最高星數寫入 `localStorage`（key: `fractionBattle`）。

### 3.2 對戰場次規則

| 項目 | 規格 |
|---|---|
| 單場最大局數 | 10 局 |
| 初始血量 | 玩家 5、怪獸 5 |
| 每局發牌 | 重新生成 |
| 提前結束 | 任一方血量歸零時，下次進入 `nextRound()` 即結算 |
| 星數保存 | 只保留每關歷史最高星數 |

## 四、畫面與資訊架構

### 4.1 開場畫面 `screenIntro`

- 遊戲標題、副標
- 宇宙怪物圖片（`picture/monster.png`，含 pulse 動畫）
- FRAKIO 劇情提示文字
- `開始冒險 🚀` 按鈕

### 4.2 腳色選擇畫面 `screenCharSelect`

- 標題「選擇你的腳色！」
- 三張腳色卡片（劍士 / 魔法師 / 弓箭手）橫向排列
- 點選後呼叫 `selectChar(name, imgPath)` → 儲存至 `state.selectedChar` → 進入關卡選擇
- RWD：手機 140px / 平板（≥600px）180px / 桌機（≥960px）230px

### 4.3 關卡選擇畫面 `screenLevelSelect`

| 關卡 | 圖示 | 名稱 | 說明 |
|---|---|---|---|
| 1 | 🪐 | 加法星球 | 分數加法 |
| 2 | ☄️ | 減法彗星 | 分數減法 |
| 3 | 🌌 | 乘法銀河 | 分數 × 整數 |
| 4 | 🕳️ | 比大小黑洞 | 大小比較 |
| 5 | ⭐ | 雙星挑戰 | 選兩張牌超越對手 |

- 每張卡片下方顯示歷史最高星數
- 第 5 關跨兩欄置中
- 提供 `← 返回` 回腳色選擇

### 4.4 對戰畫面 `screenBattle`

#### 頂部狀態列

- 左側：怪獸名稱與愛心血量
- 中央：第幾局 / 共 10 局、關卡名稱
- 右側：玩家腳色名稱與愛心血量

#### 中央主戰區

- 左側怪獸面板：`picture/monster.png`（桌機版 320px 寬）
- 中央題目區 `battleCenter`：內容依關卡動態生成

#### 底部玩家手牌區

- 左側：玩家腳色圖片（桌機版 max-width 110px）
- 右側：4 張手牌

### 4.5 結算畫面 `screenResult`

- 結果大圖示、標題、星星數
- 答對數 / 答錯數 / 剩餘血量
- 劇情式結果訊息
- `再挑戰！🔄` / `換關卡`

## 五、音效系統

### 5.1 音效檔案

| 檔案 | 用途 | 控制開關 |
|---|---|---|
| `audio/intro-bgm.mp3` | 開場三畫面 BGM（loop） | BGM |
| `audio/battle-bgm.mp3` | 對戰畫面 BGM（loop） | BGM |
| `audio/stage-clear.mp3` | 過關結算音效 / 答對音效 | BGM / SFX |
| `audio/stage-fail.mp3` | 失敗結算音效 / 答錯音效 | BGM / SFX |
| `audio/card-play.mp3` | 點擊互動元素音效 | SFX |

### 5.2 BGM 切換邏輯

| 畫面 | BGM |
|---|---|
| `screenIntro` / `screenCharSelect` / `screenLevelSelect` | `intro-bgm.mp3`（loop） |
| `screenBattle` | `battle-bgm.mp3`（loop） |
| `screenResult` | 無 BGM，播一次結算音效 |

- 切換畫面時自動停止前一段 BGM，播放對應音樂
- BGM 在同畫面間切換不重頭播（`bgmAudio.paused` 判斷）

### 5.3 全域控制按鈕

所有畫面右上角固定顯示：
- 🔊 / 🔇：音效開關（控制 card-play / correct / wrong）
- 🎵 / 🔕：BGM 開關（控制所有背景音樂與結算音效）

### 5.4 點擊音效

- 全域 `click` 事件監聽，僅在點到 `button`、`.card`、`.level-card`、`.char-card`、`.eq-slot.p-slot`、`.cmp-btn` 時播放 `audio/card-play.mp3`

## 七、卡牌系統規格

### 7.1 卡牌資料結構

```js
{
  type:  'proper' | 'improper' | 'mixed' | 'integer',
  whole: number,
  n:     number,
  d:     number,
  value: number
}
```

### 7.2 分母規格

| 情境 | 分母規則 |
|---|---|
| 關卡 1~4 | 每局隨機 `2 ~ 9` |
| 關卡 5 加法模式 | 怪獸與玩家共用 `2 ~ 5` |
| 關卡 5 乘法模式 | 怪獸分母 `2 ~ 5`；玩家另用 `2 ~ 6` |

### 7.3 答案驗證（關卡 1/2/3）

允許等值約分，禁止假分數作答：

```javascript
const sameValue   = (userTotal * ans.d === ansTotal * d); // 交叉相乘避免浮點誤差
const hasImproper = (n > 0 && n >= d);
const isCorrect   = sameValue && !hasImproper;
```

## 八、各關卡機制

| 關卡 | 說明 |
|---|---|
| 1（加法）| 選牌 → 怪獸自動出 → `[玩家] + [怪獸] = ?` → 輸入帶分數答案 |
| 2（減法）| 較大牌自動放左側 → `[大] - [小] = ?` |
| 3（乘法）| 分數牌 × 整數牌；答錯不扣玩家血（鼓勵模式）|
| 4（比大小）| 玩家先選牌鎖定，怪獸後出 → 按 ＞ ＝ ＜ 判斷 |
| 5（雙星）| 怪獸先亮牌；**+ 模式**：選兩張加總 > 怪獸；**× 模式**：選一張假分數乘以倍數（2~4）> 怪獸 |
