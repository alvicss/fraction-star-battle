# 分數星際大作戰 — 功能規格文件

**文件版本：** v2.1
**最後更新：** 2026-03-28
**規格基準：** 依 [`cld_fraction-battle.html`](./cld_fraction-battle.html) 目前實作整理  
**目標對象：** 國小中年級學生（以同分母分數概念與分數比較為主）  
**技術形式：** 單一 HTML 檔案，純前端執行，無後端依賴

## 一、產品定位

《分數星際大作戰》是一款以宇宙冒險包裝的分數練習遊戲。玩家透過選牌、計算與比較大小，和怪獸「亂亂怪」進行 10 局制對戰，在 5 個主題關卡中練習：

- 同分母分數加法
- 同分母分數減法
- 分數乘整數
- 分數大小比較
- 多步驟選牌與策略思考

遊戲強調：

- 低門檻上手
- 單機即可遊玩
- 每局重新發牌，提升變化性
- 以卡牌視覺輔助分數理解
- 用鼓勵式回饋降低答錯挫折感

## 三、整體遊戲流程

```text
開場畫面
→ 腳色選擇（選擇劍士 / 魔法師 / 弓箭手）
→ 關卡選擇
→ 對戰畫面（最多 10 局）
→ 結算畫面
→ 再挑戰 / 換關卡
```

### 3.1 全域流程說明

1. 玩家從開場進入**腳色選擇**，挑選一個代表腳色（劍士 / 魔法師 / 弓箭手）。
2. 選擇腳色後進入**關卡選擇**。（原步驟重新編號）
2. 5 個關卡全部直接開放，無解鎖限制。
3. 每場對戰初始為玩家 5 血、怪獸 5 血。
4. 每次 `nextRound()` 開始前檢查是否達成結算條件：
   - 已進行 10 局
   - 玩家血量 `<= 0`
   - 怪獸血量 `<= 0`
5. 若未結算，進入下一局並重新發牌。
6. 結算後依剩餘血量差給予 0 到 3 顆星。
7. 關卡最高星數會寫入 `localStorage`。

### 3.2 對戰場次規則

| 項目 | 規格 |
|---|---|
| 單場最大局數 | 10 局 |
| 初始血量 | 玩家 5、怪獸 5 |
| 每局發牌 | 重新生成 |
| 提前結束 | 是，當下一次進入 `nextRound()` 時若任一方血量歸零則結算 |
| 星數保存 | 只保留每關歷史最高星數 |

## 七、卡牌系統規格

### 7.1 卡牌資料結構

每張卡牌使用以下欄位表示：

```js
{
  type:  'proper' | 'improper' | 'mixed' | 'integer',
  whole: number,
  n:     number,
  d:     number,
  value: number
}
```

### 7.2 卡牌類型

| 類型 | `type` | 說明 |
|---|---|---|
| 真分數 | `proper` | `0 < n < d` |
| 假分數 | `improper` | `n > d` |
| 帶分數 | `mixed` | `whole >= 1` 且 `0 < n < d` |
| 整數 | `integer` | `whole >= 2`，`n=0, d=1` |

### 7.3 卡牌顯示規則

- 卡牌右上角顯示類型標籤
- 整數牌使用不同底色強調
- 分數採用上下排版與分數線顯示
- 帶分數顯示為「整數 + 分數」組合

標籤對應如下：

| 類型 | 顯示文案 | 標籤 class |
|---|---|---|
| `proper` | 真分數 | `badge-proper` |
| `improper` | 假分數 | `badge-improper` |
| `mixed` | 帶分數 | `badge-mixed` |
| `integer` | 整數 | `badge-integer` |

### 7.4 數值生成規則

#### 真分數 `genProperCard(d)`

- 分母使用指定 `d`
- 分子從 `1 ~ d-1` 間取與 `d` 互質的值

#### 假分數 `genImproperCard(d)`

- 分母使用指定 `d`
- 分子範圍約為 `d+1 ~ min(20, d*2 + d-1)`
- 分子需與分母互質

#### 帶分數 `genMixedCard(d)`

- 整數部分上限：`min(15, floor(19 / d))`
- 分子需與分母互質

#### 整數 `genIntegerCard()`

- 整數範圍：`2 ~ 9`

### 7.5 手牌生成規則

#### 一般玩家手牌 `genPlayerCards(d)`

- 4 張牌
- 類型從真分數 / 假分數 / 帶分數隨機產生
- 同一手牌內避免重複數值

#### 一般怪獸牌 `genFracCard(d)`

- 單張
- 使用與玩家相同分母 `d`
- 類型為真分數 / 假分數 / 帶分數隨機

### 7.6 分母規格

| 情境 | 分母規則 |
|---|---|
| 關卡 1~4 | 每局隨機 `2 ~ 9` |
| 關卡 5 加法模式 | 怪獸與玩家共用 `2 ~ 5` |
| 關卡 5 乘法模式 | 怪獸分母 `2 ~ 5`；玩家 4 張牌共用另一個分母 `2 ~ 6` |

## 八、每局共通流程

每次開新局時會依序執行：

1. 檢查是否達成結算條件。
2. 回合數 `round + 1`。
3. 重設選牌狀態、清空回饋、重置第 5 關槽位。
4. 更新血量與關卡標籤。
5. 為該局設定分母 `state.d`。
6. 依關卡生成牌組與 UI。
7. 渲染玩家手牌。

共通狀態欄位如下：

```js
const state = {
  level:             1,
  round:             0,
  playerHP:          5,
  monsterHP:         5,
  correct:           0,
  wrong:             0,
  d:                 2,
  playerCards:       [],
  monsterPlayedCard: null,
  playerPlayedIdx:   -1,
  currentAns:        null,
  phase:             'pick',
  lv5Op:             '+',
  lv5Monster:        null,
  lv5Slots:          [null, null],
  lv5Multiplier:     1
}
```

### 8.1 狀態階段 `phase`

| 值 | 說明 |
|---|---|
| `pick` | 可以點擊手牌 |
| `answer` | 已進入作答 / 判斷階段 |
| `done` | 本局完成，等待下一局 |

## 九、關卡詳細規格

## 9.1 關卡 1：加法星球

### 學習目標

- 同分母分數加法
- 帶分數與假分數換算
- 結果轉回帶分數表示

### 發牌規則

- 玩家：4 張同分母分數牌
- 怪獸：1 張同分母分數牌

### 作答流程

1. 玩家從手牌點選 1 張。
2. 玩家牌鎖定於左側槽位。
3. 怪獸牌揭曉於右側槽位。
4. 顯示等號與答案輸入欄。
5. 玩家輸入：
   - 整數部分
   - 分子
   - 分母
6. 按 `確認 ✓` 驗證答案。

### 計算規則

```text
玩家牌 + 怪獸牌
```

程式邏輯：

- 先將兩牌都轉為假分數
- 呼叫 `addFrac()`
- 再用 `toMixed()` 轉成帶分數標準答案

### 勝負規則

- 答對：怪獸 `-1 HP`
- 首次答錯：玩家 `-1 HP`
- 可按 `再試 🔄` 重答
- 重答成功不再重複扣血或加計答對次數

---

## 9.2 關卡 2：減法彗星

### 學習目標

- 同分母分數減法
- 比較兩張牌的大小
- 避免出現負數結果

### 發牌規則

- 玩家：4 張同分母分數牌
- 怪獸：1 張同分母分數牌

### 作答流程

與關卡 1 相同，但運算改為減法。

### 計算規則

```text
較大值 - 較小值
```

實作細節：

- 系統會先比較玩家牌與怪獸牌換成假分數後的分子大小
- 較大的牌放在左側槽位
- 較小的牌放在右側槽位
- 因為分母相同，所以可直接做分子相減

### 勝負規則

- 答對：怪獸 `-1 HP`
- 首次答錯：玩家 `-1 HP`
- 可重試，重試不再額外扣血

---

## 9.3 關卡 3：乘法銀河

### 學習目標

- 分數乘整數
- 將乘法結果化回帶分數

### 發牌規則

- 玩家：4 張分數牌
- 怪獸：1 張整數牌

注意：目前實作並不是「玩家同時持有整數牌與分數牌」，而是：

- 玩家只從 4 張分數牌中選 1 張
- 怪獸固定出整數牌

### 作答流程

1. 玩家點選 1 張分數牌。
2. 怪獸整數牌揭曉。
3. 玩家輸入乘積的帶分數答案。
4. 系統驗證。

### 計算規則

```text
玩家分數牌 × 怪獸整數牌
```

### 勝負規則

- 答對：怪獸 `-1 HP`
- 首次答錯：玩家不扣血
- 仍會記錄一次 `wrong`
- 可重試直到答對或進入下一局

這是明確的鼓勵式設計，降低乘法學習挫折。

---

## 9.4 關卡 4：比大小黑洞

### 學習目標

- 分數與帶分數大小比較
- 判讀 `＞ / ＝ / ＜`

### 發牌規則

- 玩家：4 張同分母分數牌
- 怪獸：1 張同分母分數牌

### 作答流程

1. 玩家先從手牌選 1 張。
2. 玩家牌放入左槽。
3. 怪獸牌揭曉於右槽。
4. 顯示三個比較按鈕：
   - `＞`
   - `＝`
   - `＜`
5. 玩家點選正確符號。

### 判定規則

系統以 `cardValue()` 比較數值：

- 若兩值差距 `< 0.0001`，視為相等
- 否則比較大小

答題後：

- 中間的 `？` 會替換為玩家選擇的符號
- 若正確，符號變為答對色
- 若錯誤，符號變為答錯色

### 勝負規則

- 答對：怪獸 `-1 HP`
- 答錯：玩家 `-1 HP`
- 此關無重試按鈕
- 直接顯示 `下一局 →`

### 重要說明

目前實作中的關卡 4 只有「選牌後判斷比較符號」這一種玩法，沒有拖曳填框或多子題模式。

---

## 9.5 關卡 5：雙星挑戰

### 學習目標

- 在條件限制下做選牌判斷
- 練習用加法或乘法讓結果超越對手
- 培養嘗試與策略切換

### 模式總覽

每局開始時隨機決定模式：

| 模式 | 機率 | 目標 |
|---|---|---|
| 加法模式 `+` | 50% | 選 2 張牌，使和 `>` 怪獸牌 |
| 乘法模式 `*` | 50% | 選 1 張牌，使該牌 × 固定整數 `>` 怪獸牌 |

---

### 9.5.1 加法模式

#### 畫面格式

```text
[玩家槽①] + [玩家槽②] > [怪獸牌]
```

#### 牌組規則

由 `genLevel5Cards('+')` 生成：

- 怪獸牌固定為帶分數
- 怪獸值約落在 `1.3 ~ 4.5`
- 玩家 4 張牌皆小於怪獸值
- 玩家牌型只會是：
  - 真分數
  - 帶分數
- 至少保證存在一組兩張牌相加後會大於怪獸牌

#### 操作方式

1. 玩家點擊手牌，依序填入兩個槽。
2. 若再次點擊已放入的牌，會取消該槽內容。
3. 兩槽都填滿後顯示 `確認 ✓`。
4. 確認後檢查兩張牌相加是否大於怪獸值。

#### 判定結果

- 成功超越：怪獸 `-1 HP`
- 未超越：玩家 `-1 HP`
- 答錯後槽位清空，玩家可重新選牌

---

### 9.5.2 乘法模式

#### 畫面格式

```text
[玩家槽] × [固定整數卡] > [怪獸牌]
```

#### 牌組規則

由 `genLevel5MulCards()` 生成：

- 怪獸牌固定為帶分數
- 固定乘數為整數 `2 ~ 4`
- 玩家 4 張牌全部共用同一分母
- 玩家牌型只會是：
  - 真分數
  - 假分數
- 不使用帶分數，避免視覺上太容易判斷
- 系統刻意保證：
  - 4 張中恰好只有 1 張是正解
  - 正解通常為假分數
  - 其餘 3 張都是錯誤干擾項

#### 操作方式

1. 玩家點選 1 張牌填入唯一槽位。
2. 再點同一張牌可取消。
3. 選定後顯示 `確認 ✓`。
4. 系統計算該牌 × 固定整數是否大於怪獸值。

#### 判定結果

- 成功超越：怪獸 `-1 HP`
- 未超越：玩家 `-1 HP`
- 答錯後自動清空槽位，讓玩家重選

## 十、答案驗證規格

## 10.1 計算型關卡（1 / 2 / 3）

驗證函式：`checkCalcAnswer(isRetry)`

### 輸入欄位

| 欄位 | 預設值處理 |
|---|---|
| 整數 `ansWhole` | 空值視為 `0` |
| 分子 `ansNumer` | 空值視為 `0` |
| 分母 `ansDenum` | 空值視為 `1` |

### 驗證流程

1. 讀取玩家輸入 `w, n, d`
2. 將玩家答案轉換為總分子 `userTotal = w * d + n`
3. 將正確答案轉換為總分子 `ansTotal = ans.whole * ans.d + ans.n`
4. 以交叉相乘比較數值是否相等：

```text
userTotal * ans.d === ansTotal * d
```

5. 額外檢查輸入是否為假分數：

```text
n > 0 且 n >= d
```

### 判定原則

| 條件 | 結果 |
|---|---|
| 數值相等且不是假分數 | 答對 |
| 數值相等但分數部分是假分數 | 答錯，顯示針對性提示 |
| 數值不相等 | 答錯，顯示一般提示 |

### 規格重點

- 允許等值但不同表示法
- 允許約分答案
- 不接受把分數部分寫成假分數
- 提示按鈕只顯示一次，按下後會隱藏

### 成功回饋

- 顯示「答對了！正確答案是 ...」
- 顯示 `下一局 →`
- 隱藏確認、再試、提示按鈕

### 失敗回饋

- 一般錯誤：提醒化成帶分數，且分子要小於分母
- 假分數錯誤：明確指出數值雖對，但分數部分格式不合規
- 隱藏確認按鈕，改顯示 `再試 🔄`

---

## 10.2 比較型關卡（4）

驗證函式：`checkLevel4(sym)`

判定基準：

- `|pVal - mVal| < 0.0001` 視為相等
- 否則看大小

回饋內容：

- 正確：`🎉 答對了！`
- 錯誤：`😅 答錯了！正確應該是 ...`

## 10.3 雙星挑戰（5）

驗證函式：`checkLevel5()`

### 加法模式

```text
cardValue(c0) + cardValue(c1) > cardValue(monster)
```

### 乘法模式

為避免浮點誤差，會先把玩家牌轉為假分數後再乘整數：

```text
(分子 × 乘數) / 分母 > 怪獸值
```

### 判定閾值

```text
玩家結果 > 怪獸值 + 0.001
```

### 失敗處理

- 記錄一次 `wrong`
- 玩家 `-1 HP`
- 清空槽位
- 讓玩家重新選牌

## 十七、與舊版規格的主要差異

本版規格特別修正以下與舊文件不一致之處：

- 關卡數為 **5 關**，不是 4 關
- 關卡 4 目前只有 **比大小按鈕模式**
- 關卡 5 才是 **雙星挑戰**
- 關卡 3 的實作是 **玩家選分數牌、怪獸出整數牌**
- 對戰結算是由 `nextRound()` 進入前統一檢查，不是答題當下立即跳轉
- 實作中保留了角色圖片位置，但目前主要仍以 emoji 顯示
